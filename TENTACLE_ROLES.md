## 三、任务分配指南

### 3.1 按任务类型分配

| 任务类型 | 首选触手 | 备选触手 |
|----------|----------|----------|
| **项目管理** | Geoff | Kia |
| **文档处理** | Eva | Anna |
| **技术开发** | Leo | Kia |
| **数据分析** | Anna | Eva |
| **设计创作** | Mia | Kia |
| **杂项任务** | Kia | Geoff |

### 3.2 按优先级分配

| 优先级 | 分配策略 |
|--------|----------|
| **高优先级** | 分配给最合适的专业触手 |
| **中优先级** | 分配给有空闲的触手 |
| **低优先级** | 分配给 Kia 或等待空闲触手 |

### 3.3 多触手协作

当任务需要多个触手协作时：

```
主脑 → 多个触手： "完成产品发布"
         ↓
Eva: 编写发布文档
Leo: 构建部署包
Mia: 设计宣传图
Geoff: 协调进度
         ↓
各触手并行执行，各自可能拆分为吸盘
         ↓
主脑汇总结果 → 主人
```

## 四、触手状态

### 4.1 状态定义

| 状态 | 说明 | 可分配新任务 |
|------|------|--------------|
| **idle** | 空闲 | ✅ 是 |
| **busy** | 执行任务中 | ❌ 否（单任务约束） |
| **paused** | 暂停（主脑离线） | ❌ 否 |
| **offline** | 离线/故障 | ❌ 否 |

### 4.2 状态转换

```
                    ┌─────────────┐
                    │    idle     │
                    │   (空闲)     │
                    └──────┬──────┘
                           │ 接收任务
                           ↓
                    ┌─────────────┐
          ┌────────│    busy     │────────┐
          │        │  (执行中)    │        │
          │        └──────┬──────┘        │
          │               │               │
    完成任务│               │主脑离线       │主脑上线
          │               ↓               │
          │        ┌─────────────┐        │
          │        │   paused    │        │
          │        │  (暂停)      │        │
          │        └──────┬──────┘        │
          │               │               │
          └───────────────┴───────────────┘
```

## 五、触手注册/注销

### 5.1 注册流程（当前手动）

```
1. 创建触手工作空间
   mkdir -p /{tentacle-id}/workspace
   mkdir -p /{tentacle-id}/workspace/tentacle-state

2. 创建 IDENTITY.md
   填写触手角色、职责等信息

3. 配置外部 Channel
   配置 webhook 地址

4. 通知主脑
   "新触手 {id} 已就绪，角色：{role}"

5. 主脑确认
   主脑将新触手加入团队列表
```

### 5.2 注销流程（当前手动）

```
1. 完成当前任务（如有）
   等待当前任务完成或暂停

2. 保存状态
   保存到 tentacle-state/ 目录

3. 通知主脑
   "触手 {id} 申请注销，原因：{reason}"

4. 主脑确认
   主脑确认后，从团队列表移除

5. 清理资源（可选）
   删除工作空间或归档
```

## 六、触手通讯

### 6.1 与主脑通讯

所有触手与主脑的通讯通过 `sessions_send` 进行，并同步到外部 Channel：

```
[触手 → 主脑] 消息内容
[触手 ← 主脑] 回复内容
```

### 6.2 消息格式

**任务接收**:
```
[{触手} → Zed] 收到任务：{任务描述}
```

**进度汇报**:
```
[{触手} → Zed] 进度：{进度描述} ({百分比}%)
```

**任务完成**:
```
[{触手} → Zed] ✅ 任务完成：{结果描述}
```

**遇到问题**:
```
[{触手} → Zed] ❌ 遇到问题：{问题描述}
```

## 七、触手工作空间规范

### 7.1 目录结构

```
/{tentacle-id}/workspace/
├── IDENTITY.md              # 触手身份信息
├── tentacle-state/          # 状态持久化目录
│   ├── current-task.json
│   ├── progress.json
│   └── context.json
├── output/                  # 输出文件目录
└── ...                      # 其他工作文件
```

### 7.2 必须文件

| 文件 | 说明 |
|------|------|
| `IDENTITY.md` | 触手身份、角色、职责定义 |
| `tentacle-state/current-task.json` | 当前任务状态（运行时） |
